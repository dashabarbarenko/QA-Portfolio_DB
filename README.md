## 📌 Description

Testing was conducted on a mobile game for the Android platform. The project included verification of gameplay mechanics, UI/UX, localization, AI systems, inventory, navigation, performance, and environmental interactions.

Test documentation was created in the form of bug reports, checklists, and test cases. Based on the testing results, a structured QA report was prepared, including analysis of identified defects, their impact on user experience, and hypotheses regarding possible root causes.

## 🧪 Types of Testing Performed

- Smoke Testing  
- Functional Testing  
- Gameplay Testing  
- UI/UX Testing  
- Localization Testing  
- Performance Testing  
- AI / NPC Testing  
- Exploratory Testing  

## 🛠️ Tools Used

- GitHub — QA portfolio and documentation storage  
- Markdown — test documentation formatting  
- Android Device Testing — testing on a real mobile device

## ✅ Tasks Performed

- Conducted exploratory testing of the game product  
- Verified gameplay mechanics and user scenarios  
- Tested UI/UX, localization, and in-game interactions  
- Identified and documented defects  
- Created bug reports according to industry standards  
- Prepared test documentation (checklists, test cases, QA report)  
- Analyzed the impact of defects on user experience and product quality  

## 📊 Conclusions

During testing, defects of varying severity were identified, including issues related to performance, localization, UI/UX, AI behavior, navigation, collisions, and gameplay mechanics.

The most critical issues were related to performance degradation, instability in shared environment scenarios, and logical inconsistencies in game systems.

Additionally, improvement suggestions were proposed for selected gameplay mechanics and overall user experience.
